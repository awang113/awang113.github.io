Motivation: Who Cares About Graduation Rates?¶

Well, as a student, me. Probably a lot of parents. And you should, too! As a society, we've agreed that education is important - graduating high school is that first step towards either higher education or starting to make one's way in the world. It's also traditionally been a huge barrier to lower income/minority groups, though over many years we've started pushing those graduation rates upwards throughout the country. Citizens who graduate with high school diplomas are more likely to vote, be employed, have higher salaries, and even live longer.

Studying graduation rates and what factors affect those rates can change the way that we provide education, hopefully fostering a better experience for students who may be struggling. Graduation rates in high schools can tell us about which demographic groups are still educationally disadvantaged and can inform decisions about where to put funding.

Here's our first little bit of code - let's import the libraries we're going to want to use for our data cleaning, exploration, and analysis down the line. We don't want to reinvent the wheel and perform complicated analyses ourselves, so we use tools curated by people much more statistics-saavy than us who have laid the groundwork! :)

The Dataset¶

Our data comes from the New York State Education Department's graduation rate database for the 2019-2020 academic school year. The data file, compiled and released in early 2021, includes information about graduation rates pre-grouped by county, district, and also distinguished by demographic features like gender and race. Usign this data set, we can pull some analysis of what factors influence high school graduation in New York's 4,772 public and charter high schools.

Let's download the CSV file provided by NYSED and load it into a pandas dataframe so that we can work with the data later on! Pandas is an extremely common and extremely useful library that contains many functions perfect for working with data. For more information about pandas, read their guide to getting started.

Data Cleaning¶

In [324]:
storesafe = data

Wowza, 227450 rows, 37 columns, and a lot of missing values - that's a big dataset! Let's start paring it down a bit. Luckily, this dataset is already quite well organized, and there are many ways to extract smaller subsets.

We do this by selecting the cohorts we want to study and by dropping all columns we won't be interested in, like year (it's all from 2019-2020), repeat information (group codes that are the same as group names), and some other extra information (like distinguishing between types of diplomas). We'll just save aggregation_index, some identifying columns, demographic indicators, and graduation rates.

We also want to mark rows that don't have relevant information. The dataset suppresses information for subgroups with small numbers (like 4 or 5) to protect student anonymity. Currently, they're marked as "-"; let's change it to NaN and drop the rows that don't have graduation data. Note that this missing data is missing because of a known factor - enrollment count for a particular subgroup. That makes it MAR data. Handling that kind of missing data is a bit outside the scope of this tutorial, but imputation is a potential solution. For now, we'll just drop the observations for the data we don't have (complete case analysis), but we should keep in mind that extremely small groups are potentially not represented in district and school-level data.

According to the README for the dataset, the "aggregation_index" column tells us what category grouping the nice people who put together the data have used. Each one is a different lens of looking at the data. They've split it as follows:

0 - Statewide

1 - Needs/Resource Capacity (measure of district's ability to meet the needs of its students - 7 subdesignations)

2 - County

3 - District

4 - School

The first grouping, statewide, is useful to have just so we know big picture statistics and can find demographic trends for the entire state of New York. We can extract that dataset and drop unnecessary columns.

The next grouping, needs/resource capacity, is one that I'm very interested in, as it categorizes students within the context of resources its districts can provide. In this way we can shed light on demographic comparisons within relative socioeconomic groupings.

The next aggregation grouping that I'm interested in is county. County is a good grouping for this data because each county's subgroups won't be so small that we have a lot of that missing, suppressed data mentioned before.

Finally, we'll also extract district data. Districts have the benefit of not being as small as the school designation but still having enough data that we can work with later when we make do our regression model. Also, districts have N/RC codes that we can use to classify, too!

Let's extract these smaller datasets so that we can do more targeted exploration late! For each dataset, we'll drop columns that are irrelevant to us.

Data Analysis and Machine Learning¶

Running data analysis can tell us whether or not certain variables really are significant in determining our dependent variable - in this case, graduation rate. I want to see if we can predict graduation rate for a particular school district (selecting district because it's not too small like school but has a decent number of data points - aruond 700).

We only have categorical variables from our dataset: sex, race, N/RC categorization - this makes analysis rather difficult. Usually, having some continuous variables with multiple possible values gives us the best kind of models. Having only categorical variables limits the amount of predictions that we can make because we have a limited set of prediction values. Still, we'll do our best!

Let's start with an least squares linear regression (yes, this is machine learning, too!), which attempts to minimize the square of residuals between observed and predicted values. This is a form of supervised machine learning, where we have a dataset and already know dependent variable outputs, but we want to create a model that might be able to predict values. Read more about the OLS regression here. Another option for this category of learning is a decision tree, where you have a flow chart to arrive at a certain class label for data, but since we want our model to guess a continuous sliding variable with many possible outcomes, we'll stick with linear regression for now!

Because we can't just feed categorical values into the model-maker (even if those categories are hiding in the form of numbers, like our N/RC categories are), we need to turn them into dummy columns of 1s and 0s. That way, our model will be able to take their values (present or not) and make decisions based on that. Note that we also need to drop one of the dummy columns per variable, because its absence or presence can be derived from the other category values.

We'll also split the data that we have into a training set and a testing set, with a ratio of 80-20. When we make models, it's a good idea to have two sets because we don't want to overfit a model by making it so attuned to a set of data that it doesn't generalize at all to other sets. Later on, we can make sure the model achieves similar results between our training and testing sets.

The independent variables that I want to use are sex, race, and N/RC category, because based on the exploration we did before, it appears that they have some bearing on graduation rates. From a linear regression, we can see how much individual variables affect graduation rate and if they're significant.

Now we have this nice summary output where statsmodels has already run the linear regression and some significance testing on the coefficients for us. You can read more about the in depth analysis of what this summary provides here, but for now let's focus on the r-squared value and the coefficients and significance of each of the variables.

The r-squared value of a predicted dataset tells us what percentage in variation our independent variables account for. On this particular random training sample, I get an r-squared value of 0.353, which loosely means that about 35% of the change seen in this dataset's graduation rates can be explained by the factors of sex, gender, or N/RC category.

Each coefficient tells us how much the predicted value changes when our population has a certain attribute. For example, we can see that being Asian/Pacific Islander boosts graduation rate predictions by around 8% compared to the "baseline" of American Indian/Alaskan Native. These values make sense when we remember the distributions over race seen in our exploratory phase. Being female apparently gives around 6% of a boost over being male, and being in a low N/RC district grants the biggest boost of all the variables, an increase of 20% over being in New York City.

The final notable column is the P>|t| column, which actually tells us whether or not a particular variable has been tested to be significant. Hypothesis testing can be confusing and used incorrectly or in a misleading way, but a very simple explanation in the context of this model is that we want to find out whether the coefficients we see are different enough from nothing that we can say that it truly does have an impact on graduation rate. A standard alpha level to base reject/accept decisions on is 0.05, so we'll use that to judge: if the p-value is below the alpha level, we can reject the null hypothesis that the variable has no effect on graduation rate. Statsmodels calculates the p-value for us, so we can just look down the list: it looks like nrc_2.0 and sub_Black have p-values well above that threshold but the others pass; it appears that their coefficients are too close to 0.

Read more in depth about hypothesis testing and what we can use it for here.

Let's think about another linear regression issue: residuals. Generating a linear model relies on certain assumptions about the data that may or may not have been met. If unmet, it calls into question the reliability of the linear model. In particular, we should find that when mapped against our independent values, our residuals are randomly scattered with no pattern. This is called homoscedasticity. If we have extreme heteroscedasticity, it can cause issues with standard error and the values that we use to do significance testing, but it would have to be quite pronounced to present a problem. We have a lot of independent values - let's take a look at how the residuals are.

Conclusion¶

From this analysis we have learned that race, gender, and need-to-resource capacity category definitely affect a district's graduation rates. We've seen from our exploration that as resources increase, gaps between demographic groups might become smaller. Although these factors alone don't explain everything about graduation rates, I suspect that if we had more related continuous data - things like the actual N/RC ratios, district incomes, number of teachers and support staff - we would be able to generate even more powerful predictive models. That's a possible further avenue: aggregating some other district-level datasets from New York that can fill in some of those missing gaps.

There are many models and datasets out there - you can explore any number of them and see if you can find interesting outcomes. Even this New York dataset, while very limited, still has more avenues of exploration because of its sheer size and inclusion of divisons - for example, we could also run the regression using county divisions and see how big of a factor county is on graduation rates (my guess is that this would be another piece of the model puzzle!).

Data science and research into which groups are in most need to additional resources can inform policy decisions to continue pushing the upward trend of graduation rates!
